## Findings considered and rejected

- **Reduce the full-text search index now**: current indexes are about 38 KB gzip per locale, too small to justify weakening body search. Reconsider with a measured payload budget as the corpus grows.
- **Centralize three post-selection expressions**: the repeated Hugo expressions are short and clear; a selector partial would add indirection without enough change pressure.
- **Adopt npm solely to manage vendored Fuse.js**: Fuse 7.0.0 has no high/critical advisory found in this audit, and a runtime package toolchain would conflict with the site's minimal design. Plan 007 adds npm only if behavioral testing is approved and keeps it test-only.
- **Treat checkout credential persistence as a primary security finding**: the workflow token is read-only in a public repository, so concrete impact is negligible. Immutable action pins provide the worthwhile CI hardening.
- **Add generic CSP/header work**: this is a static site with authored content, no authentication, no forms that mutate data, and no third-party runtime scripts. No concrete vulnerability justified the complexity.
